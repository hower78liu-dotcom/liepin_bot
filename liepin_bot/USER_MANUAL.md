# 猎聘机器人 - 用户操作指南 (USER MANUAL)

本操作手册旨在帮助具体使用者了解并正确配置数据抓取引擎所需的前置物料（账号信息、参数表）及应对极端的访问拦截。

## 1. 基础配置：`.env` 机密环境变量
首先，在项目的根目录下（即与 `main.py` 同级的目录位置）创建一个名为 `.env` 的文件。
该文件用于存储不需要且**禁止**上传到代码服务器上的核心机密：

```ini
# --- 猎聘账号与密码 ---
LIEPIN_USERNAME=您的猎聘登录手机号
LIEPIN_PASSWORD=您的访问密码

# --- 【守护者机制 (安全闸)】 ---
# 若设置为 true (推荐)，在遇到由于 IP 污染或者操作过于集中导致的非常规风控屏障时，引擎会挂起流程并强制呼唤系统通知器，提示您在可见浏览器中以真实人类视角介入干预；
# 若设置为 false，引擎在遇到无法自行识别的风控墙时会直接标记该通道死亡，并直接退出系统。
HUMAN_INTERVENTION=true

# --- （可选）打码 API 凭证 ---
# 如果您由于大批量采集遭遇弹窗滑块密集阻击，可设置云端服务器凭证以便借助第三方机器视觉实现接力计算。
JFBYM_TOKEN=

# --- 【AI 简历评审大模型服务】 ---
# 为启用 cv_matcher，请配置支持 OpenAI Chat Completions 标准协议的大模型接口密钥
API_KEY=sk-your-llm-api-key
BASE_URL=https://api.siliconflow.cn/v1
TARGET_MODEL=deepseek-ai/DeepSeek-V3.2
FREE_QUOTA_TOTAL=20000000
```

> **如何调整 AI 行为（AI 维护手册）？**
> 1. **模型切换**：在 `.env` 中修改 `TARGET_MODEL`。建议使用 `deepseek-ai/DeepSeek-V3.2` 以获得最佳性价比。
> 2. **参数微调**：若匹配度打分太宽泛，请降低 `src/modules/cv_matcher/processor.py` 中的 `temperature` 参数。
> 3. **额度监控**：系统会自动在 `data/config/ai_usage.json` 中记录已用 Token。若达到 `FREE_QUOTA_TOTAL` 设置的阈值，程序将停止请求以防超支。
> 4. **异常重试**：底层 `AIService` 已内置 3 次对 503/Rate Limit 错误的自动重试，无需人工干预。
> 5. **关闭 AI**：如果无需在清洗阶段进行归类，可在后续配置中将 `USE_AI_CLEANER` 设为 `False` （或清空 `API_KEY`）切回本地纯正则模式。

## 2. 自动化模块执行链 (Execution Pipeline)

整个招才猫架构建立在微内核机制之上。您可以按照以下声明周期逐步执行脚本：

### 第 1 步：清洗源文件
```bash
python main.py --module data_cleaner
```
**作用**：拆分处理您的 HR 主需求表单，剥离出所有包含子市的长字段（比如 `苏州, 昆山` 自动归一化为 `苏州` 且爆裂为独立条目），防范一对多混乱并导出至 `Data_Cleaned.xlsx`。

### 第 2 步：推流至搜刮引擎
```bash
python main.py --module liepin_search_preparer
```
**作用**：系统通过 `openpyxl` 将清洗完毕的列表无损化回溯合并到爬虫指令清单 `test_search.xlsx` 的对应列中。*注意：`test_search.xlsx` 模块的“职位名称”数据来源于清洗后的“搜索关键词/摘要”字段。*

### 第 3 步：启动猎聘自动化抓取
```bash
python main.py --module liepin_bot
```
**作用**：基于模拟浏览器在平台端对 `test_search.xlsx` 发出的搜索词进行无人值守的数据爬虫与防爬风控对峙，产出全量采集简历 `Get_result_leipin.xlsx`。

### 第 4 步：结果拉链回填
```bash
python main.py --module result_data_enricher
```
**作用**：使用 "(职位名称+公司名称+工作城市)" 的聚合主键将最初需求表里的`简历描述对比`及`薪资范围`无损对接映射回上述刚刚生成的 `Get_result_leipin.xlsx` 结果内。
*注意：该模块会自动在结果表中新增“岗位预算薪资”列，作为后续 AI 评审的预算参考。*

### 第 5 步：AI 大模型简历测评
```bash
python main.py --module cv_matcher
```
**作用**：多线程并发启动大语言模型。它将读取匹配回填的 JD 描述与抓取的候选人履历执行高维度 AI 机审。
*注意**：模块默认配置了本地 MD5 级别 Hash 的缓存引擎。因此若您之前计算过的岗位/简历重复出现，网络流资源将被零消耗旁路。如果想重置重计缓存，直接删掉 `data/Output/cv_match_cache.json` 即可。

## 3. 环境授权状态：`config/auth.json`
`auth.json` 是一个用于保存网站浏览器身份凭证 `Cookies` 以及 `LocalStorage` 数据的全自动缓存文件，被安全隔离在 `config/` 目录下。

- **免密工作原理**: 如果程序检测到当前路径 `config/auth.json`存在且状态未被封禁，系统将默认采用“免密静默直推登录”，极大延缓风控侦测触发。
- **失效自检**: 系统内置了 **24 小时陈旧度探测**，若凭据超过 24 小时未更新，程序会主动废弃并引导重新登录以降低风控风险。
- **如何接管更新**: 当环境由于超时被迫退回到登录框时。不用惊慌，只要在新弹出的受控浏览器窗口中**手动操作完成一次登录确认**，系统底层便会自动劫持新颁发的 Token 并静默覆盖到 `config/auth.json` 内保存起来供来日使用。
- **⚠️ 警告**: 请千万不要使用记事本去增删该文件的结构，此文件为反序列化专用。 

## 3. 守护者模式 (Guardian Mode) 人工接管
当 `HUMAN_INTERVENTION=true` 时，若自动打码引擎（CV求解器）无法通过滑块验证，系统会触发以下流程：
1. **警报唤起**: 终端控制台会打印红色警告，并触发系统蜂鸣声（BEEP）。
2. **挂起窗口**: 您有 **60 秒** 的操作有效时间。
3. **操作指导**: 您只需在弹出的浏览器中手动将滑块拖曳至正确位置。
4. **感应恢复**: 一旦您在浏览器中通过验证或进入内页，底层监控环路会立即感知，并自动更新 Session 继续执行自动化流水线，无需重启程序。

## 3. 设置抓取参数：Excel 检索驱动表
程序检索条件的物料由指定的业务数据表下发：
文件位置：`data/Input/Liepin/test_search.xlsx`

**配置准则**:
- 打开该 Excel 文件后，在里面填写的任一参数维度（如：年龄跨度、匹配职位、特殊工作年限等约束性条件）将被 `LiepinSearch_Extractor` 解包器拆解，并转换成映射于平台“高级检索面板”的填表点击指令；
- 系统底层设有了强阻断校验：如果您误删了或在运行中重命名了该文件，引擎将预检失败并主动掐断作业链路。

## 4. 浏览聚合结果
顺利度过爬取后，提纯的序列化数据会被统一投递。本项目采用了 **原子写入 (Atomic Writing)** 机制：
- **原理**: 每次保存都会先写入一个 `.tmp.xlsx` 副本，校验成功后再原子替换主文件。这有效地防止了程序崩溃导致的数据表损坏。
- **读写冲突处理**: 若输出表已打开（被 Excel/WPS 占用），系统会自动弹出 3 次锁定重试警告。

## 6. 其他容灾提示 (Disaster Recovery)
如果您在 15 秒内未关闭占用的 Excel 表格导致重试耗尽，系统会启动底层熔断保护，将本批次数据紧急落盘为临时 CSV 文件：
- **备份位置**: `src/modules/liepin_bot/backup/backup_failed_records_*.csv`
- **操作建议**: 任务结束后，请手动将该 CSV 内容合并回主表。
