# 猎聘自动化登录及滑块验证码突破技术总结

## 1. 项目概述
本项目旨在实现对猎聘网的自动化数据抓取。由于该平台部署了较为严格的安全风控策略与反爬（Anti-Bot）机制，初始登录环节成为了整个业务流程的首个技术门槛。
目前，本项目已成功基于 Python 协程与 Playwright 框架构建了一套高可用的自动登录模块，该模块能够平稳绕过浏览器特征指纹检测，并智能攻克腾讯云沙箱（TCaptcha）滑块验证码，实现稳定自动化打卡登录并完成登录态的持久化。

---

## 2. 核心执行流程

目前的自动登录（`main.py` -> `login`）工作流概括如下：

1. **环境初始化**：使用 Async Playwright 启动 Chromium，配置拟真度极高的 User-Agent、多维视口信息、时区等上下文。
2. **状态复用探测**：系统会首先检查本地是否存在有效凭证缓存（`auth.json`）。若存在，则直接携带 Cookie 和 LocalStorage 访问，从而免去重复验证，实现秒级热启动。
3. **安全注入（Anti-Detect）**：在页面加载早期注入 JS 脚本遮蔽 `navigator.webdriver` 等明显的自动化指纹驱动标志。
4. **模拟按键交互**：寻找并点击“密码登录”标签卡。通过向输入框注入数百毫秒随机延迟的敲击间隔（`random.randint(100, 200)`），最大程度拟合真实人类的键盘输入节奏填入账号与密码。
5. **智能探测安全沙箱**：提交登录表单后，系统智能挂起并持续嗅探 DOM 树中是否加载了 `#tcaptcha_iframe` 弹窗。
6. **滑块验证码突围**：一旦平台风控触发唤起滑块验证的沙箱（iframe），调用图像计算引擎与轨迹生成引擎完成滑块拼合动作（下文详述）。
7. **凭证捕捉与序列化**：验证通过并跳转到指定状态页后，捕捉完整的 Browser Context，序列化保存为 `auth.json`，供后续的数据采集或持久爬取环节共享使用。

## 3. 面临的核心难点与解决方案

在自动登录阶段，主要面临**反自动化特征检测**与**滑块验证码突破**两个维度的技术围剿。

### 难点一：Playwright/Selenium 驱动特征暴露
商业反爬系统通常会通过检测浏览器的全局对象暴露、非标准化视口或无头节点行为等判定非自然流量。
- **解决方案**：
  - 弃用容易被打标签的单纯无头模式（`headless=True`），转为开启具有真实 GPU 渲染树与全尺寸 UI 的 `headless=False`。
  - 启动参数中强行剥离特征：`args=["--disable-blink-features=AutomationControlled"]`
  - 使用 `add_init_script` 动态向 Javascript 虚拟机掩盖注入器足迹：`Object.defineProperty(navigator, 'webdriver', { get: () => undefined })`。

### 难点二：滑块缺口图像的识别提取（纯视觉层）
由于平台不定期更新验证码前端混淆模式，直接通过分析 CSS 背景图偏移值往往会陷入 DOM 加密与动态反推的泥沼。我们需要直接通过截图进行纯计算机视觉层的处理。
- **解决方案 (方案：`slider_solver.py :: find_gap_local`)**：
  - 通过注入 JS 直接获得滑块 iframe 沙箱底图的高清截图字节流。
  - 弃用传统的像素点单行遍历或弱匹配，**重构为基于 OpenCV 的边缘侦测（Canny）与几何轮廓计算法（findContours）**。
  - 提取阴影灰度特征后，算法在指定尺寸范围内（过滤背景噪点）寻找接近滑块宽高比（0.5~2.0 且面积合适）的闭合外轮廓，精准圈定缺口的候选坐标（X轴）。同时具备“边缘像素密度扫描”的防御性降级兜底方案，在特定图像失效时保障鲁棒性。

### 难点三：跨域 Iframe 尺寸塌陷与缩放映射
在某些窗口比例下，前端框架缩放了沙箱内的元素（如原图长宽比与页面 DOM 盒模型展现出来的 CSS长宽比有出入），导致计算的图像像素位移无法等比映射到物理鼠标滑动上。
- **解决方案**：
  - 脱离复杂的 Playwright 定位器比例获取法，向沙箱注入原生原生 JavaScript：`getBoundingClientRect()` ，实时、直观地同步计算并扣除背景左边距偏移与滑块按钮原点偏移，求解到最终需被执行的纯净 X轴 拖拽距离值。

### 难点四：鼠标滑动轨迹的风控模型检测
即便算准了缺口距离，直接使用 `.mouse.move` API 匀速拖过会由于其理想化的线性特征，瞬间被后端的“行为分析风控系统”判定为机器行为，并赐予“网络异常”或“请刷新重试”的屏蔽。
- **解决方案 (方案：`generate_human_track`)**：
  - 引入了**三段式仿生运动学物理算法**来生成离散的鼠标位移阵列。
  - 根据目标距离，动态演算一套包含**启动时高加速 (Acceleration) -> 中程变频匀速波动 (Volatility) -> 终局极限刹车减速与回弹校准 (Overshoot & Back-track)** 的仿生时间序列轨迹。
  - 在拖拽实现 (`perform_slide`) 中引入段间随机浮动延迟机制，以及通过随机加入 `[+1, 0, -1]` 的微小 Y 轴抖动，高度还原人手握持鼠标时的肌肉微震颤特征，彻底欺骗检测模型。

## 4. 后续探索点

当前的自动登录引擎表现稳定。后续业务流已推进至于登录后通过界面检索提取职位。由于目标检索页面DOM节点重叠和结构易变，引发了部分定位子节点的 30000ms 等待超时：
`Locator.fill: Timeout 30000ms exceeded ...`
该类问题并不属于反风控范畴，其解决策略侧重于在 DOM 元素预热加载机制上的适配。后续可以逐步加强针对业务面板渲染树的重试容错（Wait）手段。
