# 猎聘自动化抓取实施计划

本文档用于描述猎聘简历自动化检索与明细抓取流程的具体实现计划。

## 需求背景
目标：利用 Playwright 完成猎聘网模拟登录（含复杂滑块验证），根据 Excel (`D:\ljg\leipin\test_search.xlsx`) 的条件自动分批提交检索请求，并将前 20 条搜索结果（含简历明细弹窗）自动导出合并为包含多维度结构化数据的 Excel 结果文件 (`D:\ljg\leipin\Get_result\Get_result_leipin.xlsx`)。

## 方案设计思路
### 1. 结构与流程编排
我们将全自动脚本分解为三个核心模块：
- **`LiepinAuthBot` (登录认证模块)**
  基于 `AutoLogin_Ref.md` 实现。包含页面导航、账号输入及“三段式+微小抖动”的非匀速物理轨迹滑块验证。通过存储提取 Session Context 实现登录状态复用，避免每次请求均需重新认证。
- **`LiepinSearchEngine` (条件检索与解析模块)**
  读取本地测试条件 Excel，对检索页面 (`https://h.liepin.com/search/getConditionItem`) 进行表单填充（防剪贴板注入，采用逐字节模拟敲击），并进行检索条件（如学历下拉框的多重兼容选择）映射。
- **`LiepinResumeExtractor` (详情采集与存储模块)**
  抓取单次检索前 20 条记录（若不足取最大值，若为 0 单独跳过）。通过点击头像模拟进入详情页并提取求职意向、历任经历等深层信息，组装结构化字典并通过 Pandas 刷新至本地的结果 Excel 文件。

### 2. 核心难点突破与反爬对抗策略 (Anti-Bot)
1. **真实 UA 自定义**：构造逼真 User-Agent 指纹并禁用特征 (`--disable-blink-features=AutomationControlled`)。
2. **拟人性轨迹验证**：借助 OpenCV 从截图计算阴影缺口，调用贝塞尔或物理阻尼算法生成具备明显加速、匀速、减速、过冲回落等生物特征的位移数组，发送至 `page.mouse.move()`。
3. **延迟与节流**：坚决摒弃 0 延迟赋值。对关键节点执行 `random.uniform(1.5, 3.5)` 的动态等待；输入时施加 `delay=150` 的模拟敲打延迟。

## 具体任务分解 (Task Breakdown)
*详见 `task.md`*

1. **环境初始化与基础类封装**：配置 `playwright-stealth` 注入环境，规划好主入库脚本。
2. **滑动验证码核心算法编写**：完成基于背景/滑块图像截取的 OpenCV 缺口匹配函数，以及拟真拖拽轨迹生成器。
3. **验证登录流程打通**：连接真实账号（`13940861948`），完成密码注入与滑块验证，验证成功跳转并落地 Cookie。
4. **Excel 读取与搜索表单映射开发**：使用 `pandas` 解析查询源文件，根据规则将文本参数转化并勾选 Web 表单控件。
5. **候选人检索遍历流控编写**：处理翻页、点击详情、异常弹窗及防空集校验等边缘逻辑。
6. **信息组装与落地模块开发**：规范化表头合并，利用 `openpyxl` 循环追加数据，确保数据完整性并加入断点容错（Save on loop）。

## 验证与测试方案
测试流程将分为以下阶段：
1. **滑块测试**：单独执行滑块拖拽脚本模块，验证通过腾讯 TCaptcha 校验的成功率（预期成功率 >= 80%）。
2. **表单映射测试**：录入边缘测试用例（如包含本/硕/博选项组合，或极高薪范围），可视化验证 UI 层点击行为的准确性。
3. **全链路端到端跑通 (E2E)**：基于 `test_search.xlsx` 中至少两条测试记录，核对最终写入 `Get_result_leipin.xlsx` 数据中字段值的错漏与乱码情况。
