# 猎聘简历检索爬虫脚本重构任务计划表

该任务计划基于原始 Playwright Codegen 录制的代码，目标是实现基于 Excel 条件单自动化循环检索、抽取候选人详情，并严格遵循多种防止被标记为机器人的反爬策略。

## 第一阶段：环境与基础配置
- [ ] **项目环境检查**：确保 Python 环境已正确安装 `playwright`, `pandas`, `openpyxl` 等必要依赖。
- [ ] **初始目录结构验证**：确保输入文件 `D:\ljg\leipin\test_search.xlsx` 和输出目录 `D:\ljg\leipin\Get_result\` 存在及可读写。
- [ ] **配置浏览器反爬启动参数**：
  - 加载已存的 Session 状态 `auth.json` 避免脚本频繁触发登录验证码。
  - 关闭无头模式：通过设置 `headless=False` 避开对自动化工具界面的检测标记。
  - 注入真实的 `User-Agent` 信息，使得环境尽可能贴近真人用户浏览器。

## 第二阶段：登录与页面导航
- [ ] **登录页面载入**：开启浏览器并访问 `https://h.liepin.com/account/login`。
- [ ] **人工辅助登录等待**：在登录界面显式阻塞 `50` 秒，让用户有充足时间处理扫码、输入账号密码验证码、滑块验证操作。
- [ ] **跳转检索入口**：登录完毕后，导航跳转至找人搜索页面 `https://h.liepin.com/search/getConditionItem`。

## 第三阶段：读取 Excel 数据与搜索循环机制
通过循环读取 `test_search.xlsx` 文件中的数据，针对每条记录执行搜索：
- [ ] **解析并提取查询条件**：职位名称、公司名称、工作城市、经验要求、学历要求、年龄要求、性别要求、薪资范围、搜索关键词、摘要、简历描述对比、活跃度等字段。
- [ ] **字段匹配与页面录入**：
  - **职位名称**：使用键盘带延迟的模拟逐字敲击输入 (`type(text, delay=100)`)，替代瞬间粘贴的 `fill()`。
  - **工作城市**：根据读取信息进行选择（如果非空则寻找需要城市）。
  - **经验要求**：在工作年限选项中点击满足条件的对应项。
  - **学历要求（向下兼容）**：对要求学历进行层级匹配并多选。如要求本科，则勾选本科、硕士、博士/博士后；专科则勾选专科、本科、硕士、博士/博士后等以此类推。
  - **年龄与性别设定**：若有年龄限制比如 35岁以下只填写最大值 35；为空则不限。性别同样（空则不限）。
  - **其他字段**：录入对应的活跃度等标签。
- [ ] **搜索防空机制**：如果搜索结果为空集，执行 `pass` 跳过逻辑，开始处理 Excel 中的下一个岗位。
- [ ] **退出条件判断**：直到 Excel 解析出来的当前记录为空，则退出整个数据搜索主流程。

## 第四阶段：候选人详情抓取与阈值控制
对第三阶段单条非空的搜索结果列表执行简历明细项抓取：
- [ ] **安全条数限定 (阈值设定)**：设置读取单次搜索结果列表上限次数的阈值，初始值定为 **20条**。如果检索结果小于 20，则取最大值作为获取数据的结束点。
- [ ] **单条简历切入**：通过点击当前页面的 候选人名称 (name) 字段进入该候选人的详细信息页。
- [ ] **详细字段抽取**：主要抓取包含以下字段但不仅限于：
  - 候选人名称
  - 求职意向
  - 工作经历
  - 项目经历
  - 教育经历等
- [ ] **回退与断点恢复**：数据抓取完整后关闭详情页并返回主结果列表，切至下一条数据进行抽取。确保每次循环稳定无误。

## 第五阶段：数据持久化与结果保存
- [ ] **结构化组合数据**：合并本次搜索对应的检索条件（职位名称、公司名称等）、匹配度，并与第四阶段详细抽取的列表组合。
- [ ] **输出保存**：将记录持久化存入输出路径 `D:\ljg\leipin\Get_result\Get_result_leipin.xlsx`。
- [ ] **格式对齐校验**：确保导出的 Excel 表头精确包含要求的字段：职位名称、公司名称、工作城市、经验要求、学历要求、年龄要求、匹配度、候选人名称、求职意向、工作经历、项目经历、教育经历。

## 🚫 严格的反爬虫操作与稳定性规范要求
为了应对猎聘风控以及维护运行时的稳定性，编写脚本必须**100%**贯彻以下行为准则：

1. **绝对禁止全速突发点击或填充**：任意 `fill()` 或 `click()` 执行完毕，强制添加等待代码 `await page.wait_for_timeout(1000)`。
2. **拒绝瞬间粘贴式输入**：不得使用 `fill()` 直接赋文本，一律使用 `type(text, delay=100)` 发送逐字模拟击键延时事件。
3. **加入随机思考延迟**：在每个核心的输入、勾选、跳转、提交等操作循环步骤间，植入 `random.uniform(1, 3)` 秒随眠，模拟人类真实游览浏览。
4. **添加元素显式等待自动控制**：在下一步操作前必须加入自动等待逻辑，保证下拉选单、验证码弹窗、跳转元素已经充分渲染和进入 DOM，才能进行后续交互。
